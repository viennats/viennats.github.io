Examples

All examples, including input geometries, can be found in the examples folder on GitHub. These examples should introduce the user to configuration and highlight different capabilities of ViennaTS, but do not explain all possible configurations. For an extensive explanation of all models and parameters, refer to the User Guide.

Constant Deposition

This model just uses some constant normal velocities to advance the surface. It is especially useful when trying to create simple geometries quickly. The parameter file and the resulting output of this example are shown below. Note that the constant rates will keep being applied, even after the cavity is closed off, since these rates ignore visibility, while the directional rates are only applied to surface elements visible from the source plane. To stop the constant rates in a cavity, the parameter isotropic_rates should be used.

Simulation Properties:
Resolution: 200x280 grid cells
Simulation time: 20s
Files:
Download this example (zip) 
// inline comment
/*
  block comment
*/
geometry_file = "geometry2.dx";   // input geometry

output_path = "./isotropic/";  // relative to this file or absolute

omp_threads=3; // how many CPU threads to use

cfl_condition = 0.1;  // advection accuracy, must be <0.5

grid_delta = 0.5; // grid resolution (absolute)

// average surface normal for ray tracing
default_disc_orientation = {1,0,0};

num_dimensions = 2;

/*
  by defualt ViennaTS only outputs .lvst files
  .lvst output can be disabled by setting print_lvst=false;
  other filetypes can be used to:
  e.g.: vtk file output
*/
print_vtk=true;

/*
how to treat boundaries:
direction of source plane must be INFINITE
*/
boundary_conditions = {
	{INFINITE,INFINITE},
	{REFLECTIVE,REFLECTIVE}
};

open_boundary="-x"; //direction of source plane

processes = {
  {
    process_time= 200;

    model_name="ConstantRates";

    /*
      The following are process specific parameters:
    */
    parameters= {
      constant_rates={0.1}; // rate to surface normal
      //isotropic_rates={0.1};
      direction={1,0,0};  // for directional_rates
      directional_rates={0.3}; // rate in direction

    };

    // Start outputting at time t=0
    // then 100 times at intervals of dt=2
    output_times_period_length = 2;
    output_times_periodicity=100;
    output_times={0};

  }
};

SF6 Etching

Sulfur-hexafluoride is commonly used in semiconductor fabrication processes for directional etching. A model describing this process is implemented in ViennaTS and uses different particle types for ray tracing to create physically meaningful structures. This example shows the etching of Poly-Si under a mask until an etch stop layer is reached.

Simulation Properties:
Resolution: 120x120x170 grid cells
Simulation time: 15m
Files:
Download this example (zip) 
geometry_file = "Mask.vtk, Silicon.vtk";
surface_geometry=true;


print_rates=true;	// save calculated rates
print_velocities=true;	// save velocities
print_materials=true;	// save different materials
remove_bottom=false;	// keep the bottom of surfaces
print_vtk=true;		// print vtk files

cfl_condition = 0.45;

grid_delta=5e-10;

default_disk_orientation={0,0,1};

num_dimensions = 3;

omp_threads=8;

open_boundary="+z";

boundary_conditions = {
	{REFLECTIVE,REFLECTIVE},
	{REFLECTIVE,REFLECTIVE},
	{INFINITE,INFINITE}
};


processes={
	{
		model_name="SF6_CH2F2PlasmaEtching";
		active_layers = {2};
		mask_layers = {1};
		process_time=25;
		add_layer=1;

		output_times={0};
		output_times_period_length=1;
		output_times_periodicity=25;

		// these are model specific parameters
		parameters= {
			direction={0,0,-1};

			// how many particles to launch per grid point
	    		statistical_accuracy=100;

	  		min_ion_energy=50;	// ion energy
	  		delta_ion_energy=40;	// ion energy spread

          		flux_ion=1.5e15;	// these are the input fluxes
          		flux_etchant=1.3e16;
          		flux_polymer=4.5e15;

	  		temperature=298.;
		};
	}
};
