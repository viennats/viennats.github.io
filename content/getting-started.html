Getting Started

Get ViennaTS

Download a binary or build ViennaTS from source.

Running ViennaTS

ViennaTS is run in the command line by passing the parameter file as the first argument:

$ viennats par.txt

Where par.txt is the parameter file specifying all relevant information.

The general structure of the parameter file is as follows:

General parameters
(input geometry, boundary conditions, resolution,...)

processes = {
  {
    Process specific parameters
    (The model to use, output times, length of process,...)

    parameters= {
      Model specific parameters
      (strongly model dependent)

    };
  }
};

The example section contains guided examples to help getting started.

A detailed summary of all required and optional parameters can be found in the User Guide.

Viewing Output

By default, ViennaTS outputs binary levelset files (.lvst). They are direct copies of the internal data structure used by ViennaTS and the most memory efficient. For visualisation, ViennaTS can also output different other data structures, such as VTK files. These are ASCII text files containing an explicit surface representation.

Especially for long simulations, we recommend only outputting .lvst files at short intervals and converting them to VTK files later for visualisation. In this way memory consumption during simulation is kept low, keeping the maximum number of time steps. Conversion to other formats for visualisation is straight-forward using:

$ viennats -ls2vtk output.lvst

This will create a file "output.vtk", which can be viewed with a visualisation tool, such as ParaView.
